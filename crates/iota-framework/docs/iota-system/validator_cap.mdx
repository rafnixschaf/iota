## Resource `UnverifiedValidatorOperationCap`

The capability object is created when creating a new <code>
Validator</code> or when the
validator explicitly creates a new capability object for rotation/revocation.
The holder address of this object can perform some validator operations on behalf of
the authorizer validator. Thus, if a validator wants to separate the keys for operation
(such as reference gas price setting or tallying rule reporting) from fund/staking, it
could transfer this capability object to another address.
To facilitate rotating/revocation, <code>
Validator</code> stores the ID of currently valid
<code>
<Link to="validator_cap#0x3_validator_cap_UnverifiedValidatorOperationCap">UnverifiedValidatorOperationCap</Link></code>. Thus, before converting <code>
<Link to="validator_cap#0x3_validator_cap_UnverifiedValidatorOperationCap">UnverifiedValidatorOperationCap</Link></code>
to <code>
<Link to="validator_cap#0x3_validator_cap_ValidatorOperationCap">ValidatorOperationCap</Link></code>, verification needs to be done to make sure
the cap object is still valid.
