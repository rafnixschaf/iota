<Link id="0x2_dynamic_field"/>
In addition to the fields declared in its type definition, a Iota object can have dynamic fields
that can be added after the object has been constructed. Unlike ordinary field names
(which are always statically declared identifiers) a dynamic field name can be any value with
the <code>
<b>copy</b></code>, <code>
drop</code>, and <code>
store</code> abilities, e.g. an integer, a boolean, or a string.
This gives Iota programmers the flexibility to extend objects on-the-fly, and it also serves as a
building block for core collection types

throws <code>

or throws <code>
<Link to="../iota-framework/dynamic_field#0x2_dynamic_field_EFieldTypeMismatch">EFieldTypeMismatch</Link></code> if the type does not match,
and may also abort with <code>
<Link to="../iota-framework/dynamic_field#0x2_dynamic_field_EBCSSerializationFailure">EBCSSerializationFailure</Link></code>
